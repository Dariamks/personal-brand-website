## 职业背景

我是一名 Java 后端开发工程师，目前专注于工业物联网平台的后端架构与高并发数据接入。具备从零搭建 TCP/MQTT 数据接入服务、设计告警分析引擎以及性能调优的实战经验。对 Spring 生态、微服务架构、消息队列与数据库优化有深入的实践和理解。

## 职业故事

我的编程之路始于大学软件工程专业。从最初的数据结构与算法，到后来的 Java Web 开发，我逐渐发现后端系统的魅力——它能承载真实世界的业务逻辑，处理海量数据，并直接影响用户的体验。

毕业后，我加入了一家专注于智慧矿山领域的科技公司。在那里，我第一次独立负责从设备数据接入、协议解析到告警分析、可视化大屏的全链路后端开发。面对日均 200W+ 的设备上报数据，我从索引优化、SQL 重写、缓存策略到分表方案，一步步将系统响应速度提升 10 倍以上。这段经历让我深刻体会到：**好的后端工程师不仅要会写接口，更要能从数据流、性能瓶颈和业务需求三个维度同时思考问题。**

我享受从底层原理去理解一项技术——无论是 Spring IoC 容器的启动流程、MySQL 索引的 B+Tree 结构，还是 Netty 的事件循环模型。我相信，只有真正理解技术是如何工作的，才能在关键时刻做出正确的架构决策。

## 价值观

- **代码即文档** — 好的命名、清晰的结构和合理的注释比任何外部文档都更有价值
- **量化驱动** — 不用"感觉快了"来评价优化，用响应时长、QPS、命中率等指标来衡量
- **原理优先** — 从底层原理出发理解技术，而非停留在"会用就行"的层面
- **持续沉淀** — 通过博客和技术笔记将碎片化经验转化为可复用的方法论
